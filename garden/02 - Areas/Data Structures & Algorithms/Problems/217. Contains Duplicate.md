# Contains Duplicate

## Problem Statement

Given an integer array `nums`, return `true` if any value appears at least twice in the array, and return `false` if every element is distinct.

>[!Example]+ Example 1
>**Input**: `nums = [1,2,3,1]`
>**Output**: `true`
>**Explanation**:
>The element 1 occurs at the indices 0 and 3.

>[!Example]+ Example 2
>**Input**: `nums = [1,2,3,4]`
>**Output**: `false`
>**Explanation**:
>All elements are distinct.

>[!Example]+ Example 3
>**Input**: `nums = [1,1,1,3,3,4,3,2,4,2]`
>**Output**: `true`

>[!warning]+ Constraints
>
>- `1 <= nums.length <= 10^5`
>
>- `-10^9 <= nums[i] <= 10^9`
>

## Problem Breakdown

1. **Understand the problem**:
   - Input: Array of integers (nums)
   - Output: Boolean (true if any duplicate exists, false otherwise)
   - Constraints: Array length 1-10^5, values -10^9 to 10^9

2. **Initial thoughts**:
   - We need to check if any value appears more than once
   - Naive approach: Check every pair of elements (nested loops)
   - Example: [1,2,3,1] → true because 1 appears twice

3. **Pattern recognition**:
   - This is about finding duplicates → tracking "seen elements" pattern
   - When checking for existence/duplicates → hash table/set is usually efficient
   - Similar to problems like Two Sum, where we track what we've seen

4. **Solution strategy**:
   - Use a hash set to track elements we've seen so far
   - Hash set gives O(1) lookup, which is more efficient than O(n) linear search
   - Time complexity: O(n), Space complexity: O(n)

5. **Implementation plan**:
   - Initialize an empty set
   - Iterate through the array once
   - For each element, check if it's in the set
   - If yes, return true (found duplicate)
   - If not, add to set and continue
   - If we finish the loop, return false (no duplicates)
